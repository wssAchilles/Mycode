## 🚨 常见问题解决

### 1. 端口被占用
如果5000端口被占用，可以修改启动脚本中的端口：
```python
app.run(debug=True, host='0.0.0.0', port=8000)
```

### 2. 数据库问题
如果遇到数据库错误，删除`app.db`文件重新运行：
```bash
del app.db
python start.py
```

### 3. 依赖包问题
重新安装依赖：
```bash
pip install -r requirements.txt --force-reinstall
```

### 4. 权限问题
确保有文件写入权限，特别是logs和uploads目录。
